# 维度 5：功能规格与字段定义（Functional Spec & Field Definition）

> 评估 PRD 中页面、字段、接口、能力的描述是否清晰到可以直接进入研发。这是研发返工率最高的维度。

**权重：14%（最高）**

---

## 评估问题清单（共 13 题）

### P0（致命）

| ID | 问题 | 判定标准 | 命中证据示例 |
|---|---|---|---|
| **D5-Q1** | 是否给出了页面 / 接口 / 能力的详细规格说明？ | 每个功能点有"功能描述 + 输入 + 输出 + 行为" | 缺失：仅列功能名 |
| **D5-Q2** | 表格中每个字段是否有明确的含义说明？ | 字段表中每行包含「字段名 + 含义」 | 缺失：仅有字段名无含义 |
| **D5-Q3** | 字段是否说明了数据来源（接口 / 系统 / 计算公式）？ | 显式说明"取 CIS 平台"、"通过借据号查询资金平台"等 | 缺失：字段从天而降，研发不知去哪取 |

### P1（重要）

| ID | 问题 | 判定标准 | 命中证据示例 |
|---|---|---|---|
| **D5-Q4** | 字段是否标注了数据类型 / 枚举值 / 格式（日期格式、精度等）？ | 标注 "日期格式 yyyy-MM-dd"、"金额精度 2 位"、"枚举：A/B/C" | 缺失：金额字段未说明精度 |
| **D5-Q5** | 字段是否说明必填 / 选填？ | 显式标注 必填 / 非必填 / 条件必填 | 缺失：表单字段必填关系不明 |
| **D5-Q6** | 列表 / 搜索栏字段是否说明默认值、排序规则、分页？ | 列表说明"按操作时间倒序"、搜索默认空、分页方式 | 缺失：列表行为靠研发猜 |
| **D5-Q7** | 操作按钮是否说明权限 / 可见条件 / 二次确认弹窗？ | 每个按钮说明"谁可见"、"什么状态可点"、"是否需要确认" | 缺失：删除按钮无二次确认 |
| **D5-Q8** | 计算类字段是否给出公式示例？ | 公式可计算（如 fee1+fee2+催费+我方罚息+资方罚息+提前结清费） | 缺失：仅说"汇总计算"无公式 |
| **D5-Q9** | 是否提供了原型图 / 设计稿 / 示例数据？ | 关键页面有截图或链接 | 缺失：复杂页面无任何视觉参考 |

### P2（建议）

| ID | 问题 | 判定标准 | 命中证据示例 |
|---|---|---|---|
| **D5-Q10** | 字典表 / 枚举值是否单独整理在附录？ | 文档末尾或独立章节有"附录：字典表" | 缺失：枚举值散落各处难维护 |
| **D5-Q11** | 字段命名是否符合团队规范（驼峰 / 下划线 / 前缀）？ | 与既有系统命名风格一致 | 命中：风格混乱（snake 与 camel 混用） |
| **D5-Q12** | 是否说明字段长度 / 最大值 / 最小值的约束？ | 文本类有长度上限、数值类有范围 | 缺失：无长度限制可能引发存储问题 |
| **D5-Q13** | 是否区分了"对内字段"和"对外字段"？ | 外部接口字段单独列出且说明可见性 | 缺失：内部字段误暴露给外部 |

---

## 评分要点

- 本维度权重 14% 是 10 个维度最高，因为这里的缺失最直接导致研发返工与上下游联调失败。
- **D5-Q3（字段来源）** 在跨系统 PRD 中尤其关键：优秀案例《资产债转凭证》中每个字段都标注"取 CIS"、"取金融机构管理页面"。
- **D5-Q8（计算公式）** 在金融账务类 PRD 中是高发短板：含糊的"应还其他费用总额"必须列出 fee1+fee2+催费+... 的具体加项。
- **D5-Q4 + D5-Q12** 联动：金额字段未说精度可能导致 0.01 元差异引发资金风险。
